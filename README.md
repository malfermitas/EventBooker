# EventBooker 📅🎟️

**EventBooker** — это современный сервис для организации мероприятий и бронирования билетов на них, написанный на языке Go (v1.25) с использованием чистой архитектуры и мощного стека технологий.

## 🚀 Описание проекта
Сервис позволяет пользователям создавать события с ограниченной вместимостью, управлять ими и бронировать места. Реализован механизм временного бронирования (TTL), при котором неоплаченные брони автоматически аннулируются по истечении заданного времени.

## ✨ Основные возможности
- **🔐 Аутентификация:** Полноценная система регистрации и входа на основе JWT (Access & Refresh токены).
- **📅 Управление событиями:** Создание, просмотр и получение подробной информации о мероприятиях.
- **🎟️ Умное бронирование:** 
  - Резервирование мест на события.
  - Ограничение по вместимости (capacity).
  - Автоматическая отмена просроченных бронирований (Booking TTL).
  - Подтверждение бронирований.
- **🌐 Фронтенд:** Встроенный интерфейс на Go HTML Templates + Vanilla JS/CSS.
- **📢 Уведомления:** Интеграция с сервисом уведомлений (notifier) и поддержка Telegram.
- **🏗️ Надежность:** Использование транзакций БД с механизмом повторных попыток (retries).

## 🛠 Технологический стек
- **Язык:** [Go 1.25+](https://go.dev/)
- **HTTP фреймворк:** [Gin Gonic](https://gin-gonic.com/) (через библиотеку [wbf](https://github.com/wb-go/wbf))
- **База данных:** [PostgreSQL 16](https://www.postgresql.org/)
- **Драйвер БД:** [pgx/v5](https://github.com/jackc/pgx)
- **Аутентификация:** JWT (JSON Web Tokens)
- **Логирование:** slog, zerolog, zap (поддержка нескольких движков)
- **Конфигурация:** [cleanenv](https://github.com/ilyakaznacheev/cleanenv)
- **Контейнеризация:** Docker & Docker Compose

## 🏁 Быстрый запуск

### 🐳 Через Docker (рекомендуется)
Самый простой способ запустить всё необходимое — использовать Docker Compose:

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/your-repo/eventbooker.git
   cd eventbooker
   ```
2. Запустите проект:
   ```bash
   docker-compose up --build
   ```
3. Приложение будет доступно по адресу: [http://localhost:8080](http://localhost:8080)

### 💻 Локальный запуск
1. Убедитесь, что у вас установлен **Go 1.25** и запущен **PostgreSQL**.
2. Создайте базу данных и примените миграции из `db/schema/schema.sql` (или используйте `migrate` для файлов в `db/migrations`).
3. Настройте параметры подключения в `configs/config.yaml`.
4. Запустите сервер:
   ```bash
   go run cmd/server/main.go
   ```

## 🔗 Основные API эндпоинты

### Аутентификация
- `POST /auth/register` — регистрация нового пользователя.
- `POST /auth/login` — вход и получение JWT токенов.
- `POST /auth/refresh` — обновление Access-токена.
- `GET /me` — информация о текущем пользователе.

### События
- `GET /events` — список всех мероприятий.
- `POST /events` — создание нового мероприятия.
- `GET /events/:id` — детальная информация о мероприятии.

### Бронирование
- `POST /events/:id/book` — резервирование места.
- `POST /events/:id/confirm` — подтверждение бронирования.

## ⚙️ Конфигурация
Настройки приложения хранятся в `configs/config.yaml`. Основные секции:
- `app.http`: Настройки HTTP сервера (порт, таймауты).
- `app.postgres`: Параметры подключения к БД (DSN).
- `app.auth`: Настройки JWT (секрет, TTL токенов).
- `app.notifier`: Параметры интеграции с сервисом уведомлений.
